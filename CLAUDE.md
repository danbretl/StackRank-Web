## What it is

A web app for **stack-ranking** things — movies first, but meant to generalize. You add a movie and the app slots it into your ordered list via **binary-insertion comparisons** (pick the one you like better, repeatedly, until its exact position is found). Around that core sit suggestions, watch/skip queues, a movie-detail pane, and a rich shareable poster generator.

An experimental **Books** vertical slice now lives at `/books`: it is noindex, device-only, and
deliberately excludes account sync, public snapshots, and raster cover exports while Open Library
provider/artwork policy is evaluated. The cross-domain plan is in
`notes/feature-ideas/multi-domain-expansion-plan.md`.

As of July 16, 2026, Books expansion and provider outreach are paused. **StackRank Dogs is the active
next-category initiative** and now has a comprehensive local-first product at `/dogs`: a generated
VBO catalog, editorial packs, ranking/lists/backup/export flows, and responsive browser coverage.
Its public launch remains gated by rights-approved artwork plus approval/application of the additive
sync migration. The authoritative plan is `notes/feature-ideas/dogs-launch-plan.md`; exact status and
release gates live in `notes/testing/dogs-implementation-status.md`.

- **Live movies app:** https://www.stackrankapp.com/movies (Vercel; `stackrankapp.com` redirects to `www`, and the `www` root temporarily redirects to `/movies` until a cross-category home ships)
- **Legacy host:** https://danbretl.github.io/StackRank-Web/ remains available during the origin migration so users can recover any browser-local data.
- **Repo:** https://github.com/danbretl/StackRank-Web (local dir is `stackrank`, remote repo is `StackRank-Web`)
- **Design intent (from day one):** simple, clean, slick, **monochrome**, punchy, modern. Minimal header; adding/ranking is front and center.

## Architecture

Plain **static single-page app — no build system, no framework, no bundler, no npm dependencies for the app itself.** Core browser files:

- **`index.html`** — markup; loads `app.js` as an ES module (needed for the Supabase client import). References assets with cache-busting query strings (`app.js?v=N`, `styles.css?v=N`). Its `<head>` also carries the **social link-preview metadata** (Open Graph + Twitter `summary_large_image` + canonical) pointing at `assets/og-preview.png` — the 1200×630 card generated by `scripts/build-og-image.cjs` (`npm run build:og`); bump the image meta `?v=N` after regenerating.
- **`privacy.html`** — standalone public policy and Credits page at `/privacy`; covers browser storage, optional sync, service providers, bounded telemetry, user controls, and required TMDB attribution using the approved unmodified `assets/tmdb-logo.svg`.
- **`books.html` / `books.js` / `books.css`** — private-preview Books route. It ranks Open Library
  works through the shared entity/rank-session/list/backup primitives, stores only under the Books
  namespace, and has its own focused real-browser regression flow.
- **`dogs.html` / `dogs.js` / `dogs.css`** — public-route-ready, local-first Dogs product. It loads
  the generated static VBO catalog and editorial packs under `data/dogs/`, stores provider-qualified
  breed identities only in Dogs keys, and keeps sync/public/raster capabilities off until their
  separate persistence and artwork gates pass.
- **`home.html` / `home.js` / `home.css`** — noindex family-home release artifact. It presents Movies
  and Dogs equally and reads count-only local progress without loading either app bundle. It is not
  wired to `/`; the authorized temporary root redirect still points to `/movies`.
- **`shared.html` / `shared.js`** — static read-only public shared-list route for `/s/:slug`. It uses the publishable Supabase client to fetch an active `shared_lists` snapshot and renders a monochrome poster-grid ranking, tappable movie-detail sheets, and a "Rank your own movies" CTA; metadata is generic because there is no per-slug SSR.
- **`app.js`** (~10200 lines) — application wiring and DOM-bound logic. It imports the vendored Supabase browser bundle from `vendor/supabase-js-2.108.2.js?v=1`; update `notes/supabase-js-vendor.md` when upgrading it.
- **`lib/`** — DOM-free ES modules extracted from `app.js` and shared by the browser app plus tests (`zip`, `text`, `format`, `movie`, `persistence`, `pack-progress`, `insights`, `taste`, `tonight`, `packs`, `suggestions`, `share-options`, `share-export`, `share-svg`, `share-link`, `fullscreen-ranking`, `ranking`, `review`, `undo`, `backup`, `telemetry`, `ftue`, `auth`, `app-shell`, `lists`) plus cross-domain primitives (`category`, `entity`, `rank-session`, `ranked-list`, `category-backup`, `catalog`, `provider-policy`, `category-lists`, `category-ranking`, `category-remote-persistence`, category descriptors under `lib/categories/`, and Dogs-specific pure helpers in `lib/dogs.js`).
- **`tests/`** — zero-dependency Node test suite for the `lib/` modules.
- **`styles.css`** — all styling.

**Movie identity is the TMDB id (`tmdbId`)**, with a normalized `title + year` fallback for legacy items that predate id tracking.

**Persistence** (priority: *never lose data*):
- Signed out → `localStorage`. Signed in → **Supabase**, with localStorage as fallback/merge.
- On load the app **merges** local + remote by id and keeps the newest (`{ movies, updated_at }` timestamp guard) so an older server snapshot can't shrink a larger local list.
- If that no-loss merge appends movies that exist only in an older snapshot, the app surfaces a toast explaining they were added to the bottom and offers **Review order**.
- Every ranking save updates localStorage even when the Supabase write succeeds. Destructive operations (clear/import/restore) therefore cannot leave an older local snapshot behind for merge-on-load to resurrect.
- Signed-out users get an actionable **Download backup / Sign in to sync** nudge every 25 local rankings, rate-limited to about 30 days, and immediately after browser storage starts failing; the nudge waits until comparison/review UI has settled so it does not hide the active choice.
- localStorage keys: `stackrank:movies:v1` (ranking), `stackrank:suggestion-queues:v1` (watch/skip queues), `stackrank:share-options:v1`, `stackrank:inspired-seed:v1`, `stackrank:backup-nudge:v1`.
- Dogs uses isolated local keys: `stackrank:dogs:ranking:v1`, `stackrank:dogs:queues:v1`,
  `stackrank:dogs:pack-progress:v1`, `stackrank:dogs:ranking-view:v1`, and a per-tab
  `stackrank:dogs:app-destination:v1`. Dogs backups are category-bound and never read or replace
  Movies or Books payloads.
- The active Rank / Ranking / You destination is a per-tab convenience stored in `sessionStorage` as `stackrank:app-destination:v1`. Meaningful activity refreshes it at most once per minute; after 30 minutes without activity it expires back to Rank. It never affects user data persistence.

**Supabase** (project ref `hrfhakrxsllrqmscxxpb`, URL + public publishable key live in `app.js`):
- Tables: **`rankings`** (`list_id` PK, `movies` jsonb, `updated_at`), **`movie_lists`** (one row per `(list_id, list_type)`; `list_type` ∈ `watch` / `not_interested`; upsert `onConflict: "list_id,list_type"`), **`suggestion_packs`** + **`pack_progress`**, **`shared_lists`** (public read-only snapshots at `/s/:slug`, owner-managed by `list_id`, revoked rows hidden from anon/public reads while owners can still manage them), and insert-only **`product_events`** for privacy-bounded anonymous funnel telemetry. User JSON payloads are bounded in Postgres and mirrored client-side before remote writes: ranking/queue `movies` ≤ 1 MB each, shared-list `payload` ≤ 1 MB, pack-progress `state` ≤ 8 KB. Product events retain raw rows for 180 days via Supabase Cron and reject more than 500 rows for one `session_id`.
- The unapplied migration `20260716090037_add_category_data_tables.sql` prepares additive
  `category_rankings`, `category_lists`, `category_pack_progress`, and `category_shared_lists`
  tables with explicit grants, payload bounds, and category/owner RLS. It does not alter mature
  Movies tables. Do not apply it without explicit authorization and the database probes in
  `notes/testing/dogs-supabase-rls-review.md`.
- Per-user list id is `user:<auth.uid()>`. RLS policies are scoped to `list_id = 'user:' || (select auth.uid())` (the `(select …)` wrap is a Supabase RLS perf requirement — keep it). Migrations live in `supabase/migrations/`.
- **Edge functions** (Deno/TS in `supabase/functions/`); the Movies functions proxy TMDB so the
  TMDB key stays server-side (Supabase secret `TMDB_API_KEY`):
  - `tmdb-search` — autocomplete (returns `tmdbId`, year, poster).
  - `tmdb-suggest` — `type=popular | recommendations | essentials`.
  - `tmdb-detail` — runtime/genres/director/cast/overview for the detail pane.
  - `tmdb-image` — **public, CORS-`*`** poster proxy with a coarse per-IP token bucket and long immutable poster caching; exists so PNG export can draw posters onto a `<canvas>` (cross-origin TMDB images can't be rasterized directly).
  - `tonight-pick` — enrichment + mood scoring for the "Pick something for tonight" helper: takes up to 80 queue `ids` plus an optional free-text `mood` (≤160 chars), enriches each candidate from TMDB (runtime/genres/keywords/director/top-3 cast; per-instance 12h cache; never returns vote data), interprets the vibe through the curated lexicon in `_shared/mood.ts` (Deno-tested in `mood_test.ts`), and returns per-candidate mood-fit scores with the matched senses/keywords/era. Carries a per-IP token bucket (60 req / 5 min) because each request fans out to TMDB.
  - `books-search` — experimental Books autocomplete proxy. It makes identified, low-volume Open
    Library searches and returns only normalized work id/title/authors/year/cover id; origin,
    publishable-key, query-size, timeout, cache, and per-IP rate guards mirror the Movies boundary.
  - Function auth configuration lives in `supabase/config.toml`. Search/suggest/detail/tonight-pick run with the platform JWT gate off, reject disallowed browser `Origin`s via `supabase/functions/_shared/http.ts`, and validate the `apikey` header against `SUPABASE_PUBLISHABLE_KEYS` in `supabase/functions/_shared/publishable-key.ts`; image remains intentionally public.
- **Auth:** a dedicated **sign-in view** (`#signin-overlay`, opened from the header "Sign in" and the settings popover) always offers email **magic link** and supports **Google OAuth** / **Apple OAuth** once those providers are enabled in Supabase. The view reads Supabase's public Auth settings and hides unavailable OAuth methods, so partially configured providers never become broken user-facing buttons. OAuth uses `supabase.auth.signInWithOAuth`; magic link uses `signInWithOtp`. Magic-link email is delivered via **Resend** SMTP (sender `stackrank@danbretl.com`); branded production templates and subjects mirror `supabase/templates/` and were applied through the Supabase Management API. Pure helpers (redirect URL, email validation, provider availability, sign-out warning copy) live in `lib/auth.js`. **OAuth provider secrets are never committed** — `config.toml` reads them via `env(...)`; production Google/Apple still require their provider-console credentials (see `notes/feature-ideas/auth-upgrade.md`). Auth is **origin-scoped** — sessions and localStorage on `danbretl.github.io`, `stackrankapp.com`, and `localhost` are separate; users must sign in again on each origin, while paths on the same origin share the session and browser data. Signing out confirms that the account copy stays intact while the current device shows an empty local list until the user signs in again. `file://` breaks magic links entirely. The hosted Supabase Site URL is `https://www.stackrankapp.com/movies`; the old production roots, apex `/movies`, StackRank Vercel previews, legacy GitHub Pages URL, and local development URLs remain in the redirect allowlist. Auth init is guarded by `AUTH_INIT_TIMEOUT_MS` (3200ms) and falls back to the local list if Supabase is unreachable.

## Feature map (what exists)

- **StackRank Dogs:** `/dogs` searches 1,264 selectable VBO-derived breeds/types through canonical
  names and aliases, then uses the shared binary-insertion mechanics with undo/cancel. It includes
  Recently ranked, 46 validated editorial packs, Detailed/Photos/Compact ranking views, safe facets,
  pointer/touch/keyboard reorder, Review order, provenance-rich details, Curious about / Not for me,
  rank-weighted Taste patterns, category backup/restore/name import, and text/Markdown/JSON exports.
  Missing photos use a neutral fallback. Account sync, public links, and raster sharing are visibly
  disabled until their additive database and rights-purpose gates are approved.

- **Experimental Books preview:** `/books` offers Open Library work search, three starter shelves,
  binary-insertion ranking, reorder/remove, simple stats, and category-bound backup/restore. It is
  noindex and local-only; starter shelves remain usable if search fails. Its stored entities use
  provider-qualified work ids and cannot read/write the Movies namespace.

- **App shell:** the primary destinations are **Rank / Ranking / You**. Rank is a full-width action launchpad: a prominent single-input **Rank Bar** directly below navigation, a compact Recently ranked rail, six rotating movie packs on the open page canvas, and three ruled lanes of individual movie prompts. Ranking owns the ordered-list artifact. You is a modular personal space for progress, Tonight, Taste Explorer, and Watch next; Hidden remains secondary management. Desktop and iPad use the same width-driven composition; iPhone keeps the same destinations in a fixed bottom navigation and turns discovery rows into horizontal touch shelves. Refreshing within an active per-tab session restores the current destination; after 30 minutes without activity, the shell defaults back to Rank.
- **Add a movie:** TMDB autocomplete only (no free-text adds). The search field is an ARIA combobox wired to the suggestions listbox (`aria-expanded`, `aria-controls`, `aria-activedescendant`, option `aria-selected`). Clicking a suggestion starts ranking immediately — no confirm button.
- **First-run quick start:** an empty ranking gets a compact inline explanation, three varied starter packs (Fan Favorite Core / Studio Ghibli / Black Cinema), and direct title import; after the first movie it collapses to "add one more," then disappears at two. Pure state/selection rules live in `lib/ftue.js`; privacy-bounded events measure exposure, pack/import adoption, first comparison, and two/five-movie activation. Review targets and the canonical query live in `notes/feature-ideas/first-run-quick-start.md`.
- **Ranking:** binary-insertion comparison (the *opening* comparison's starter movie is jittered around the list midpoint by ~18% of the list length via `firstComparisonIndex` in `lib/ranking.js`, so back-to-back rankings don't always lead with the same middle title — at most 1–2 extra comparisons; every narrowed range still uses the exact midpoint); **Undo last choice** / **Cancel ranking** (only one shows at a time; cancel restores the movie + the pre-ranking scroll position). The native Ranking page has remembered **Detailed / Posters / Compact** views plus one **Display & filters** panel; there is no separate fullscreen mode or jump-to-rank control. Reordering is disabled while filtered. Custom **pointer-based drag** reorder (handle-only on touch/pen, drag-anywhere with a mouse) plus keyboard reorder from focused Move handles with Arrow Up / Arrow Down and live rank announcements works in all three views. Touch-capable screens share one interaction contract: item taps open detail, rare actions live in overflow, and explicit Move mode exposes handles; phone-size layouts use a compact View/Move/Share header. Dragging has a movement threshold, scroll-owner-aware auto-scroll, no-op detection, cancellation on pointer interruption/Escape, and Undo after a committed move. **Ranking review mode** (quiet Review order link, disabled with <2 movies) is an adjacent-pair audit pass: `buildReviewQueue` (`lib/review.js`) picks pairs favoring the just-added movie, then most-recently-ranked (`rankedAt`), then an even spread, capped at 8; it reuses the comparison panel + `is-comparing` layout (adding `is-reviewing` for the Keep/Swap/End controls), persists swaps immediately, and the whole session is undoable via one end-of-session toast (Escape ends it).
- **Movie packs:** 114 curated fallback packs across 22 categories. The populated Rank launchpad shows six packs selected afresh on each page load (and on explicit refresh): up to two active continuations, at most one organic head start, then category-diverse unfinished packs. This prevents progress from freezing the shelf while keeping packs first-class; the empty first-run experience still uses its three deliberate starter packs. **View all packs** has a compact expandable filter panel with text search (pack metadata, movie titles, years/decades), category selection, and progress-state chips (Not started / In progress / Head start / Complete / Updated). Pack detail mirrors All Packs with the same icon-only **×** close control in the upper-right, while optional Prev / Pack x of y / Next navigation forms one compact group aligned to the toolbar's left edge; it supports Rank/Save/Hide, bulk Save all/Hide all, and continuous Rank all mode. Progress is derived from ranking/queues and persisted locally/Supabase when available.
- **Suggestions:** three sections — *Inspired by [movie]* (rank-weighted seed from your top 10), *All-time essentials* (TMDB discover, curated filters), *Popular now*. Each section explains its source, and each movie card gets a concise async-enriched reason: distinctive shared genre or ranked-seed provenance for Inspired, decade/genre for Essentials, and TMDB popularity/genre for Popular. Pending reason lines stay hidden until detail enrichment settles; TMDB ratings are never shown. Each card has **Save** (→ Watch next) and **Hide** (→ Not for me) plus an info button. `SUGGESTION_PAGE_SIZE = 3`; per-section refresh; hidden during ranking.
- **Queues:** *Watch next* and *Not for me* (synced via `movie_lists`). Watch next is a You widget; Not for me is reached through the secondary **Manage hidden movies** action. Rows click to rank; move/remove; info button. On mobile, queue rows keep Rank visible and move Info, Save/Hide, and Remove into overflow. Subtitles show live counts.
- **Pick something for tonight:** a collapsed "Decision helper" panel at the top of the Watch next pane (shown with ≥2 queued movies with TMDB ids). The user picks a time window (Any / Under 90m / Around 2h / Over 2½h) and optionally types a free-text mood/vibe; the `tonight-pick` edge function enriches the queue and scores the vibe server-side, then `lib/tonight.js` blends mood fit with rank-weighted taste affinity, soft-edged runtime fit, and queue-age freshness into three explained picks (diversity-guarded; "Show different picks" rotates unshown candidates). The status line echoes how the vibe was read; "Watch this" collapses to a single card with Rank it (→ normal comparison flow, `ranking_started` source `tonight`) / Change pick. Nothing is persisted on the user's behalf; the slate recomputes when a shown movie leaves the queue and window changes re-rank instantly from the cached run. Falls back to detail-cache enrichment (no mood) when the function is unreachable. Design record: `notes/feature-ideas/pick-something-for-tonight.md`.
- **Movie detail pane:** opened by the info icon (card tap still ranks). Actions adapt to source (suggestion: Rank/Save/Hide; Watch next: Rank/Hide/Remove; Not for me: Rank/Save/Remove). Tapping the poster opens it full-res (TMDB `original`) in the **shared lightbox** (see Share Studio).
- **Recently ranked:** a compact orientation rail immediately below the Rank Bar; it links directly into the ranking without competing with discovery.
- **Taste Explorer:** after five ranked movies, the You widget lazily enriches the first 120 movies and shows up to three rank-weighted recurring signals (genre, era, director/cast). Selecting one reveals the exact ranked movies behind it; evidence rows open detail, **Open this ranking lens** opens Posters as a rank-preserving filtered subset on the native Ranking page, and genuinely matching pack metadata can open a focused pack search. Pure rules live in `lib/taste.js`; no derived state is persisted.
- **Share Studio** — *the* share flow: a configurable **generated SVG poster → PNG (via canvas)**, plus Markdown/JSON/Text exports (all driven by the same options), plus signed-in-only **public snapshot links**. Toggleable sections (top/bottom picks, eras, genres, cast & crew, saved/hidden, **packs**, whole list with posters-only / text-only / posters+text), **tones** (Factual/Punchy/Funny/Extreme) and **looks** (Classic/Cinema/Marquee/Color Pop/Warm). `SHARE_OPTIONS_VERSION = 7` (migrates saved prefs; v7 added the `packs` toggle, default on). The link controls publish/update/copy/revoke a short `/s/:slug` snapshot containing only `{ displayName, movies:[{ title, year, posterPath, tmdbId }] }`; it is not live sync, and signed-out users see a sign-in prompt. The image exports also have a **Format** (Single image / Image set) and a **Shape** (**Skinny** / **Wide**): the image set emits a header + grouped cards (top&bottom, eras, genres, cast&crew, **saved&hidden = 5 most-recent of each, by `savedAt`/`hiddenAt`**, packs, whole-list paginated with counters) sized to content (max 1200×2600); a 2+-card set downloads as a single **.zip** (`createStoredZipBlob`, a hand-rolled no-deps stored-ZIP writer) and the studio preview labels each card with a numbered caption pill. The **packs** section (between saved/hidden and whole list) shows a 4-up meta strip (completed / in-progress / movies-ranked / most-explored category) over up to four in-app-style pack cards (2×2 poster collage, title, subtitle, status, progress bar, ✓ when complete); driven by `getSharePackSummary()`/`getSharePackFeatured()`/`sharePackCardStatus()` over derived `getPackStats`, and self-hides with no pack engagement (its Include toggle is labelled "Movie packs"). **Empty-section handling is general:** once detail loading settles, any section with no content is omitted from every export and its Include toggle is disabled with a "… (empty)" label — `shareSectionAvailability()` computes per-section content presence (genres/cast+crew stay available while async detail data is still loading) and `updateShareIncludeAvailability()` drives the toggle disable/label. Image builders live in a shared layer — `buildShareImages()` dispatches; `shareSectionBuilders()` returns width-agnostic section descriptors; `buildShareSvg` (Skinny), `buildShareWideSvg` (Wide: non-list sections in a 2-col masonry, then the whole list full-width below at 4/4/11 cols for mixed/text/posters), and `buildShareImageSetPages` compose them; Shape only applies to Single image. The single-image **saved/hidden** section also lists the 3 most-recent of each. Internally `shape` is `"skinny"|"wide"` (migrated from the older `"portrait"|"landscape"`). Stats use passive metadata: `rankedAt` on rank, `savedAt`/`hiddenAt` on queue moves (no DB schema change). **Lightbox:** tapping a preview (single SVG or an image-set card) opens it in a shared full-res overlay (`#share-lightbox`, JS `openShareLightbox`/`renderShareLightbox`/`closeLightbox`) — native pinch-zoom + pan, tap-to-toggle enlarge, dismiss via ×/backdrop/Esc. In share mode (`.is-share`) it carries a **Download/Share action bar** wired to the same per-image export funcs (`downloadCurrentShareSetPage`/`downloadSharePng`, `shareNativePng`); for image sets (`.is-set`) it adds **prev/next nav + arrow keys + horizontal swipe (only when not zoomed) + an "i/total CAPTION" counter**, kept in sync with `shareSetPageIndex`. The same overlay is reused chrome-free by the **detail-pane poster** via `openPosterLightbox` (TMDB `original` res, no bar/nav).
- **List settings:** a quiet gear icon in the page header opens a popover with account status, a **Sign in** button (opens the dedicated sign-in view — see Auth), Sign out, Clear rankings, and **Backup & import**. On mobile the header stays a single 44px-tall row with the Movies subtitle beneath the brand, and the settings panel is fixed to the viewport edges so it cannot open off-screen. Download backup emits a versioned JSON snapshot of ranking, queues, pack progress, and Share Studio options; Restore backup replaces those surfaces exactly after confirmation and is undoable. **Import titles** accepts an ordered one-title-per-line list (optional years), matches through `tmdb-search`, and auto-resolves each line via `chooseAutomaticTmdbMatch` (in `lib/backup.js`) which returns `{ movie, confidence }`: `"exact"` for a unique normalized title (+year) match, or `"guess"` for one disambiguated by popularity (TMDB returns most-popular first), by article/punctuation-insensitive comparison ("Death of Stalin" ↔ "The Death of Stalin"), or by a 2+-word subset match ("Sonic 3" → "Sonic the Hedgehog 3"). Guesses are pre-selected but flagged "Best guess — review"; only truly unmatched lines stay unresolved. Replacing a non-empty ranking still requires explicit confirmation. Queues are preserved except for movies now present in the imported ranking. The pack detail reached via **View all packs** has a separate "Pack x of y" prev/next pager (buttons + left/right arrow keys) that steps through the currently filtered/sorted browser list and is hidden in the browser view itself.
- **Debug panel:** append `?debug=1` to the URL.
- **Product instrumentation:** Vercel Web Analytics collects first-party pageviews on `www`; custom product events go to `product_events` with an ephemeral per-page UUID, strict event/property allowlists, bucketed counts, production-only collection, DNT/GPC opt-out, and no movie/account identifiers. Production `?debug=1` and `navigator.webdriver` visits disable both streams so QA does not contaminate product decisions; always use `?debug=1` for rendered production smoke. The clean analysis cutoff is `2026-06-29 18:44:36+00`. Pure guards live in `lib/telemetry.js`; event definitions and queries are documented in `notes/feature-ideas/product-instrumentation.md`. Taste Explorer records only explorer/lens opens, never the selected signal; public share-link events record publish/view counts without slugs, URLs, titles, or ids.
- **Privacy & credits:** `/privacy` is linked from the app footer and sign-in view. It describes local/account data handling and deletion controls, carries TMDB's approved logo and required non-endorsement notice, and documents the experimental Books storage/Open Library requests and credit.

## Styling

- **Design tokens** (`:root`): monochrome — `--bg-ink:#111`, `--bg:#f2f2f2`, `--panel:#fff`, `--ink-soft:#6b6b6b`, `--border:#111`, `--border-soft:#e2e2e2`. Font is **Space Grotesk** (Google Fonts); radial-gradient page background; panels are white / 20px radius / soft shadow. `[hidden]{display:none!important}` is a global rule several flows rely on.
- **Action hierarchy:** high-value primary actions use a white editorial double-keyline treatment with a heavier label and a restrained gray hover/lift on fine pointers; secondary actions use one outline; tertiary actions stay borderless. Solid black is reserved for selected/pressed controls, progress, content surfaces, and irreversible confirmation rather than routine CTA fills. The Add/Search field follows the same primary keyline at every size while preserving its phone pill geometry and 48–56px touch height.
- **Body state classes** drive takeover and interaction modes: `is-comparing`, `is-detail-open`, `is-share-open`, `is-dragging`.
- **Responsive system:** available width/height controls composition; input capability controls interaction. `max-width:720px` covers phone portrait, while `(any-pointer:coarse) and (orientation:landscape) and (max-height:600px)` gives modern phone landscape the same mobile shell/takeover behavior even when its CSS width exceeds 720px. `min-width:721px`/`min-width:900px` govern larger compositions, so desktop and same-size iPad geometry stay aligned. Hover affordances are gated behind `@media (hover:hover) and (pointer:fine)`; touch/pen targets and semantics use `@media (any-pointer:coarse)`. Do **not** use pointer capability to choose columns/order, and do not put tap-critical styling in `:hover`.

## Design & product taste (read before touching UI)

- **Mobile is first-class** (it's used heavily on phones): big thumb-friendly tap targets, both comparison choices must fit one screen in **portrait *and* landscape**, don't waste vertical space, keep reachable controls low.
- **Motion is minimal, consistent, and predictable** — one-shot animations, no jitter.
- When unsure about a layout/typography choice, **mirror the primary ranking list.**
- **"Favorite"/taste signals are rank-weighted** (where a movie sits in the list matters), *not* raw frequency. Charts can show counts, but callouts say "highest ranked …".
- **Don't surface TMDB ratings** — they bias users away from their own taste; StackRank is personal preference, not consensus.
- After finishing/canceling a ranking, **`blur()` the title input, never `focus()`** — focusing pops the mobile keyboard. (This bug has recurred; watch for it in any "after ranking" path.)
- In the share poster: titles must keep inner **padding** and never touch borders (width-based wrapping + clip-path backstop); names never truncate.

## Debugging gotchas

- **"My change isn't showing up"** → first suspect (a) uncommitted/unpushed work (Vercel deploys from GitHub `main`) or (b) browser/edge cache (hard-refresh, and check the `?v=` cache key) — *not* a logic bug.
- **Blank page locally / no data** → usually a JS boot error halting `app.js` before render, or simply being signed out on `localhost` (origin-scoped session). Check the console first.
- **PNG export has no posters** → must go through the `tmdb-image` proxy and account for SVG group `translate()` transforms when drawing onto canvas.
