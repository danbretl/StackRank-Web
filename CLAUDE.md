## What it is

A web app for **stack-ranking** things ‚Äî movies first, but meant to generalize. You add a movie and the app slots it into your ordered list via **binary-insertion comparisons** (pick the one you like better, repeatedly, until its exact position is found). Around that core sit suggestions, watch/skip queues, a movie-detail pane, and a rich shareable poster generator.

- **Live:** https://www.stackrankapp.com/ (Vercel; `stackrankapp.com` redirects here)
- **Legacy host:** https://danbretl.github.io/StackRank-Web/ remains available during the origin migration so users can recover any browser-local data.
- **Repo:** https://github.com/danbretl/StackRank-Web (local dir is `stackrank`, remote repo is `StackRank-Web`)
- **Design intent (from day one):** simple, clean, slick, **monochrome**, punchy, modern. Minimal header; adding/ranking is front and center.

## Architecture

Plain **static single-page app ‚Äî no build system, no framework, no bundler, no npm dependencies for the app itself.** Three core files:

- **`index.html`** ‚Äî markup; loads `app.js` as an ES module (needed for the Supabase client import). References assets with cache-busting query strings (`app.js?v=N`, `styles.css?v=N`). Its `<head>` also carries the **social link-preview metadata** (Open Graph + Twitter `summary_large_image` + canonical) pointing at `assets/og-preview.png` ‚Äî the 1200√ó630 card generated by `scripts/build-og-image.cjs` (`npm run build:og`); bump the image meta `?v=N` after regenerating.
- **`app.js`** (~8300 lines) ‚Äî application wiring and DOM-bound logic.
- **`lib/`** ‚Äî DOM-free ES modules extracted from `app.js` and shared by the browser app plus tests (`zip`, `text`, `format`, `movie`, `persistence`, `insights`, `packs`, `share-export`, `share-svg`, `ranking`, `review`, `undo`, `backup`).
- **`tests/`** ‚Äî zero-dependency Node test suite for the `lib/` modules.
- **`styles.css`** ‚Äî all styling.

**Movie identity is the TMDB id (`tmdbId`)**, with a normalized `title + year` fallback for legacy items that predate id tracking.

**Persistence** (priority: *never lose data*):
- Signed out ‚Üí `localStorage`. Signed in ‚Üí **Supabase**, with localStorage as fallback/merge.
- On load the app **merges** local + remote by id and keeps the newest (`{ movies, updated_at }` timestamp guard) so an older server snapshot can't shrink a larger local list.
- Every ranking save updates localStorage even when the Supabase write succeeds. Destructive operations (clear/import/restore) therefore cannot leave an older local snapshot behind for merge-on-load to resurrect.
- localStorage keys: `stackrank:movies:v1` (ranking), `stackrank:suggestion-queues:v1` (watch/skip queues), `stackrank:share-options:v1`, `stackrank:inspired-seed:v1`.

**Supabase** (project ref `hrfhakrxsllrqmscxxpb`, URL + public publishable key live in `app.js`):
- Tables: **`rankings`** (`list_id` PK, `movies` jsonb, `updated_at`) and **`movie_lists`** (one row per `(list_id, list_type)`; `list_type` ‚àà `watch` / `not_interested`; upsert `onConflict: "list_id,list_type"`).
- Per-user list id is `user:<auth.uid()>`. RLS policies are scoped to `list_id = 'user:' || (select auth.uid())` (the `(select ‚Ä¶)` wrap is a Supabase RLS perf requirement ‚Äî keep it). Migrations live in `supabase/migrations/`.
- **Edge functions** (Deno/TS in `supabase/functions/`), all proxy TMDB so the TMDB key stays server-side (Supabase secret `TMDB_API_KEY`):
  - `tmdb-search` ‚Äî autocomplete (returns `tmdbId`, year, poster).
  - `tmdb-suggest` ‚Äî `type=popular | recommendations | essentials`.
  - `tmdb-detail` ‚Äî runtime/genres/director/cast/overview for the detail pane.
  - `tmdb-image` ‚Äî **public, CORS-`*`** poster proxy; exists so PNG export can draw posters onto a `<canvas>` (cross-origin TMDB images can't be rasterized directly).
  - Function auth configuration lives in `supabase/config.toml`. Search/suggest/detail run with the platform JWT gate off and validate the `apikey` header against `SUPABASE_PUBLISHABLE_KEYS` in `supabase/functions/_shared/publishable-key.ts`; image remains intentionally public.
- **Auth:** Supabase email **magic link**, delivered via **Resend** SMTP (sender `no-reply@danbretl.com`). Auth is **origin-scoped** ‚Äî sessions and localStorage on `danbretl.github.io`, `stackrankapp.com`, and `localhost` are separate; users must sign in again on each origin. `file://` breaks magic links entirely. The Supabase Site URL is `https://www.stackrankapp.com/`; the apex domain, legacy GitHub Pages URL, and local development URLs remain in the redirect allowlist. Auth init is guarded by `AUTH_INIT_TIMEOUT_MS` (3200ms) and falls back to the local list if Supabase is unreachable.

## Feature map (what exists)

- **Add a movie:** TMDB autocomplete only (no free-text adds). Clicking a suggestion starts ranking immediately ‚Äî no confirm button.
- **Ranking:** binary-insertion comparison (the *opening* comparison's starter movie is jittered around the list midpoint by ~18% of the list length via `firstComparisonIndex` in `lib/ranking.js`, so back-to-back rankings don't always lead with the same middle title ‚Äî at most 1‚Äì2 extra comparisons; every narrowed range still uses the exact midpoint); **Undo last choice** / **Cancel ranking** (only one shows at a time; cancel restores the movie + the pre-ranking scroll position). Custom **pointer-based drag** reorder (handle-only on touch, drag-anywhere on desktop). **Ranking review mode** (Review icon in the ranking header, disabled with <2 movies) is an adjacent-pair audit pass: `buildReviewQueue` (`lib/review.js`) picks pairs favoring the just-added movie, then most-recently-ranked (`rankedAt`), then an even spread, capped at 8; it reuses the comparison panel + `is-comparing` layout (adding `is-reviewing` for the Keep/Swap/End controls), persists swaps immediately, and the whole session is undoable via one end-of-session toast (Escape ends it). The **full-screen ranking** is an interactive responsive poster grid with detail access, re-rank/remove, title/year filtering, Comfortable/Compact density, jump-to-rank, arrow/Home/End navigation, and undoable 2-D pointer drag (disabled while filtered).
- **Movie packs:** 114 curated fallback packs across 22 categories. The front-page shelf shows three status-prioritized packs; **View all packs** has a compact expandable filter panel with text search (pack metadata, movie titles, years/decades), category selection, and progress-state chips (Not started / In progress / Head start / Complete / Updated). Pack detail supports Rank/Save/Hide, bulk Save all/Hide all, and continuous Rank all mode; progress is derived from ranking/queues and persisted locally/Supabase when available.
- **Suggestions:** three sections ‚Äî *Inspired by [movie]* (rank-weighted seed from your top 10), *All-time essentials* (TMDB discover, curated filters), *Popular now*. Each section explains its source, and each movie card gets a concise async-enriched reason: distinctive shared genre or ranked-seed provenance for Inspired, decade/genre for Essentials, and TMDB popularity/genre for Popular. Pending reason lines stay hidden until detail enrichment settles; TMDB ratings are never shown. Each card has **Save** (‚Üí Watch next) and **Hide** (‚Üí Not for me) plus an info button. `SUGGESTION_PAGE_SIZE = 3`; per-section refresh; hidden during ranking.
- **Queues:** *Watch next* and *Not for me* (synced via `movie_lists`). Rows click to rank; move/remove; info button. Subtitles show live counts.
- **Movie detail pane:** opened by the info icon (card tap still ranks). Actions adapt to source (suggestion: Rank/Save/Hide; Watch next: Rank/Hide/Remove; Not for me: Rank/Save/Remove). Tapping the poster opens it full-res (TMDB `original`) in the **shared lightbox** (see Share Studio).
- **Recently ranked** snapshot panel (front page).
- **Share Studio** ‚Äî *the* share flow: a configurable **generated SVG poster ‚Üí PNG (via canvas)**, plus Markdown/JSON/Text exports (all driven by the same options). Toggleable sections (top/bottom picks, eras, genres, cast & crew, saved/hidden, **packs**, whole list with posters-only / text-only / posters+text), **tones** (Factual/Punchy/Funny/Extreme) and **looks** (Classic/Cinema/Marquee/Color Pop/Warm). `SHARE_OPTIONS_VERSION = 7` (migrates saved prefs; v7 added the `packs` toggle, default on). The image exports also have a **Format** (Single image / Image set) and a **Shape** (**Skinny** / **Wide**): the image set emits a header + grouped cards (top&bottom, eras, genres, cast&crew, **saved&hidden = 5 most-recent of each, by `savedAt`/`hiddenAt`**, packs, whole-list paginated with counters) sized to content (max 1200√ó2600); a 2+-card set downloads as a single **.zip** (`createStoredZipBlob`, a hand-rolled no-deps stored-ZIP writer) and the studio preview labels each card with a numbered caption pill. The **packs** section (between saved/hidden and whole list) shows a 4-up meta strip (completed / in-progress / movies-ranked / most-explored category) over up to four in-app-style pack cards (2√ó2 poster collage, title, subtitle, status, progress bar, ‚úì when complete); driven by `getSharePackSummary()`/`getSharePackFeatured()`/`sharePackCardStatus()` over derived `getPackStats`, and self-hides with no pack engagement (its Include toggle is labelled "Movie packs"). **Empty-section handling is general:** once detail loading settles, any section with no content is omitted from every export and its Include toggle is disabled with a "‚Ä¶ (empty)" label ‚Äî `shareSectionAvailability()` computes per-section content presence (genres/cast+crew stay available while async detail data is still loading) and `updateShareIncludeAvailability()` drives the toggle disable/label. Image builders live in a shared layer ‚Äî `buildShareImages()` dispatches; `shareSectionBuilders()` returns width-agnostic section descriptors; `buildShareSvg` (Skinny), `buildShareWideSvg` (Wide: non-list sections in a 2-col masonry, then the whole list full-width below at 4/4/11 cols for mixed/text/posters), and `buildShareImageSetPages` compose them; Shape only applies to Single image. The single-image **saved/hidden** section also lists the 3 most-recent of each. Internally `shape` is `"skinny"|"wide"` (migrated from the older `"portrait"|"landscape"`). Stats use passive metadata: `rankedAt` on rank, `savedAt`/`hiddenAt` on queue moves (no DB schema change). **Lightbox:** tapping a preview (single SVG or an image-set card) opens it in a shared full-res overlay (`#share-lightbox`, JS `openShareLightbox`/`renderShareLightbox`/`closeLightbox`) ‚Äî native pinch-zoom + pan, tap-to-toggle enlarge, dismiss via √ó/backdrop/Esc. In share mode (`.is-share`) it carries a **Download/Share action bar** wired to the same per-image export funcs (`downloadCurrentShareSetPage`/`downloadSharePng`, `shareNativePng`); for image sets (`.is-set`) it adds **prev/next nav + arrow keys + horizontal swipe (only when not zoomed) + an "i/total CAPTION" counter**, kept in sync with `shareSetPageIndex`. The same overlay is reused chrome-free by the **detail-pane poster** via `openPosterLightbox` (TMDB `original` res, no bar/nav).
- **List settings:** a quiet gear icon in the page header opens a popover with account status / sign in / sign out, Clear rankings, and **Backup & import**. On mobile the header stays a single 44px-tall row with the Movies subtitle beneath the brand, and the settings panel is fixed to the viewport edges so it cannot open off-screen. Download backup emits a versioned JSON snapshot of ranking, queues, pack progress, and Share Studio options; Restore backup replaces those surfaces exactly after confirmation and is undoable. **Import titles** accepts an ordered one-title-per-line list (optional years), matches through `tmdb-search`, and auto-resolves each line via `chooseAutomaticTmdbMatch` (in `lib/backup.js`) which returns `{ movie, confidence }`: `"exact"` for a unique normalized title (+year) match, or `"guess"` for one disambiguated by popularity (TMDB returns most-popular first), by article/punctuation-insensitive comparison ("Death of Stalin" ‚Üî "The Death of Stalin"), or by a 2+-word subset match ("Sonic 3" ‚Üí "Sonic the Hedgehog 3"). Guesses are pre-selected but flagged "Best guess ‚Äî review"; only truly unmatched lines stay unresolved. Replacing a non-empty ranking still requires explicit confirmation. Queues are preserved except for movies now present in the imported ranking. The pack detail reached via **View all packs** has a "Pack x of y" prev/next pager (buttons + left/right arrow keys) that steps through the currently filtered/sorted browser list and is hidden in the browser view itself.
- **Debug panel:** append `?debug=1` to the URL.

## Styling

- **Design tokens** (`:root`): monochrome ‚Äî `--bg-ink:#111`, `--bg:#f2f2f2`, `--panel:#fff`, `--ink-soft:#6b6b6b`, `--border:#111`, `--border-soft:#e2e2e2`. Font is **Space Grotesk** (Google Fonts); radial-gradient page background; panels are white / 20px radius / soft shadow. `[hidden]{display:none!important}` is a global rule several flows rely on.
- **Body state classes** drive the major modes: `is-comparing`, `is-detail-open`, `is-share-open`, `is-dragging`.
- **Responsive breakpoints**: `max-width:720px` (mobile portrait ‚Äî the comparison "ballot"), `max-width:900px and (orientation:landscape)`, `min-width:721px`/`min-width:900px` (desktop). Hover affordances are gated behind `@media (hover:hover) and (pointer:fine)` and touch sizing behind `@media (pointer:coarse)` ‚Äî so **don't put tap-critical styling in `:hover`.**

## Running it & dev workflow

- **Local server** (from repo root): `python3 -m http.server 8000` ‚Üí http://localhost:8000/ . Any port is fine; add the exact local URL to Supabase Auth redirect URLs if you need sign-in to work locally. **`file://` won't work** for auth.
- **Phone/tablet testing:** `http://<Mac-LAN-IP>:<port>/` (get IP via `ipconfig getifaddr en0`); device on same network.
- **Validate before committing / handing off:** `npm run verify`. This runs `npm test`, `node --check app.js`, `deno check --no-lock supabase/functions/*/index.ts`, and `npm run test:e2e`.
- **Automated tests:** `npm test` (zero-dep, built-in `node --test`; ~0.3s). It prints the spec reporter to the terminal and saves a timestamped report under `reports/runs/<timestamp>/` (gitignored) with `junit.xml`, `output.log`, `summary.json`, and `summary.md`; `reports/latest` symlinks to the newest run. `npm run test:e2e` launches headless Chrome/Chromium through a zero-dep CDP harness, seeds deterministic localStorage, exercises the real static app (persistence, queue ranking, undo/cancel restore, Share Studio preview, backup restore/title import, suggestion explanations, full-screen ranking interactions/drag, and mobile pack layout), and saves `reports/e2e/runs/<timestamp>/` plus `reports/e2e/latest` with summaries, JUnit, and screenshots. Set `CHROME_PATH` if Chrome is in a nonstandard location. CI uploads both unit and E2E report runs as artifacts. Pure logic is extracted into native ES modules under **`lib/`** ‚Äî `zip`, `text`, `format`, `movie` (identity/merge), `persistence` (local payload parsing, timestamp precedence, no-loss merges, queue normalization), `insights` (rank-weighted engine), `packs`, `suggestions`, `fullscreen-ranking`, `share-export`, `share-svg`, `ranking` (binary-insertion), `undo`, and `backup` ‚Äî imported by **both** `app.js` (browser, no build step) and **`tests/*.test.js`**. To make new logic testable, extract it into a DOM-free `lib/` module and have `app.js` import it (keep a thin wrapper that binds live state). Strategy, phases (0‚Äì6 plus persistence hardening done; Phase 7 browser smoke growing) and conventions: **`notes/testing/automated-tests.md`**.
- **Test expectation for future work:** new functionality should include focused tests for the new pure logic or regression risk before handoff. If a change touches ranking identity/merge, insight scoring, packs, share exports/SVG, ZIP/export behavior, or binary insertion, add or update the relevant `tests/*.test.js` file. If logic is trapped in `app.js`, extract the smallest DOM-free piece to `lib/` rather than testing through globals. UI-only changes still need a browser smoke of the affected flow; use `npm run screenshots` when responsive visual behavior is the risk.
- **CI:** GitHub Actions runs `npm run verify` on pushes to `main` and on pull requests.
- **Deploy an edge function:** `supabase functions deploy <name>` (Supabase CLI via Homebrew; project is linked to ref `hrfhakrxsllrqmscxxpb`). **You must redeploy after changing a function or its response shape.**
- **Screenshots:** `npm run screenshots` (headless Chrome; flags `--label=`, `--only=desktop-comparison,mobile-comparison-portrait,...`). Archives to `debug/screenshots/runs/<timestamp>/` + `latest/` (both gitignored).
- **Social preview image:** `npm run build:og` regenerates `assets/og-preview.png` (1200√ó630) from the design embedded in `scripts/build-og-image.cjs` via headless Chrome. After regenerating, bump the `?v=N` on the `og:image`/`twitter:image` meta tags in `index.html` so unfurler caches refresh.
- **Hosting:** Vercel project `stackrank` is connected to the GitHub repo and deploys `main` to production. Cloudflare Registrar/DNS owns `stackrankapp.com`; both apex and `www` use DNS-only CNAME records to Vercel, with Vercel issuing the apex ‚Üí `www` 308 redirect and TLS certificates. GitHub Pages remains enabled temporarily as a browser-local-data recovery path.
- **Cache-busting:** when you change JS or CSS, **bump `app.js?v=N` / `styles.css?v=N` in `index.html`** ‚Äî otherwise browsers or an edge cache can serve stale assets. Current: `app.js?v=123`, `styles.css?v=82`. **Note:** `app.js` imports some `lib/` modules with their own `?v=N` (e.g. `lib/backup.js?v=2`, `lib/packs.js?v=2`, `lib/persistence.js?v=1`, `lib/fullscreen-ranking.js?v=1`, `lib/review.js?v=1`, `lib/ranking.js?v=2`), and fetches `data/suggestion-packs.json?v=N` (constant `PACK_FALLBACK_PATH`) ‚Äî bump that query too when you change one of those files, since bumping only `app.js?v=` won't refresh a cached `lib/` import or data file.

## Conventions

- **Commits:** short, imperative, **single line, no body, no trailer** (e.g. "Replace list snapshot with recent rankings", "Sync suggestion queues with Supabase"). Commit **directly to `main`**. **Only commit/push when explicitly asked.** Stage only the relevant app/function files.
- **Feature-note follow-through:** after an explicitly requested commit/push of product changes, review `notes/feature-ideas/` and this shared brief for affected behavior, status, metrics, and follow-ups. Make any appropriate documentation updates and commit/push them as a separate follow-up documentation commit under the same authorization; do not include unrelated note rewrites.
- **Never commit:** `.DS_Store`, `debug/`, `supabase/.temp/`, `.vscode/`, `package-lock.json` (all gitignored). **Never put secrets in the repo** (TMDB key, Resend key, Supabase secret/service key). The Supabase *publishable* key in `app.js` is public-by-design and gated by RLS.
- **`notes/feature-ideas/`** holds fully-fleshed-out ideas and shipped design records (movie-detail, better-suggestion-explanations, fullscreen ranking, ranking-review-mode, undo-history-for-list-changes, personal-stats-and-taste-profile). Add to it rather than scattering TODOs.
- **Session recap text (Claude w/ iMessage):** when a *large* development session wraps (multiple features/fixes or substantial multi-step work), text Dan a concise summary via the iMessage tool once verification is done. His number is intentionally kept out of this public repo ‚Äî it lives in Claude's private memory (`text-summary-on-large-sessions`). Small one-off changes don't need a text.

## Design & product taste (read before touching UI)

- **Mobile is first-class** (it's used heavily on phones): big thumb-friendly tap targets, both comparison choices must fit one screen in **portrait *and* landscape**, don't waste vertical space, keep reachable controls low.
- **Motion is minimal, consistent, and predictable** ‚Äî one-shot animations, no jitter.
- When unsure about a layout/typography choice, **mirror the primary ranking list.**
- **"Favorite"/taste signals are rank-weighted** (where a movie sits in the list matters), *not* raw frequency. Charts can show counts, but callouts say "highest ranked ‚Ä¶".
- **Don't surface TMDB ratings** ‚Äî they bias users away from their own taste; StackRank is personal preference, not consensus.
- After finishing/canceling a ranking, **`blur()` the title input, never `focus()`** ‚Äî focusing pops the mobile keyboard. (This bug has recurred; watch for it in any "after ranking" path.)
- In the share poster: titles must keep inner **padding** and never touch borders (width-based wrapping + clip-path backstop); names never truncate.

## Debugging gotchas

- **"My change isn't showing up"** ‚Üí first suspect (a) uncommitted/unpushed work (Vercel deploys from GitHub `main`) or (b) browser/edge cache (hard-refresh, and check the `?v=` cache key) ‚Äî *not* a logic bug.
- **Blank page locally / no data** ‚Üí usually a JS boot error halting `app.js` before render, or simply being signed out on `localhost` (origin-scoped session). Check the console first.
- **PNG export has no posters** ‚Üí must go through the `tmdb-image` proxy and account for SVG group `translate()` transforms when drawing onto canvas.
